# Feishu Docx Engine 0.1.1 / Feishu Markdown Sync 0.6.3 Recovery Checklist

## Recovery reason

- `v0.6.2` is immutable and remains tagged.
- Its workflow stopped at artifact-manifest verification before either npm publish step.
- The source tree was identical, but the manifest used Node 23.11.0/npm 10.9.2 while CI used Node 24.18.0/npm 11.18.0.
- Registry checks confirmed `feishu-docx-engine@0.1.1` and `feishu-md-sync@0.6.2` were not published.
- The first Recovery PR live gate exposed an eventual-consistency window where Feishu accepted an image-to-Whiteboard replacement before the unique Whiteboard block appeared in structural readback; recovery retries only that exact unchanged readback and never repeats the write.

## Release identity

- Engine: `feishu-docx-engine@0.1.1` retained because it is still unpublished.
- CLI: `feishu-md-sync@0.6.3`.
- CLI dependency: exactly `feishu-docx-engine: 0.1.1`.
- Git tag: `v0.6.3`, created only from the merged Recovery PR commit.
- Manifest: `.github/releases/v0.6.3.json`.
- Artifact toolchain: Node 24.18.0 and npm 11.18.0.
- Engine provenance: new publication bound to `refs/tags/v0.6.3` and the tag commit.

## Generate the artifact manifest

Install dependencies and generate the manifest only through the pinned toolchain:

```bash
npx --yes --package node@24.18.0 --package npm@11.18.0 -- \
  npm ci --ignore-scripts --no-audit --no-fund
npx --yes --package node@24.18.0 --package npm@11.18.0 -- \
  npm run release:manifest:write
npx --yes --package node@24.18.0 --package npm@11.18.0 -- \
  npm run release:manifest:check
```

The script must pack engine and CLI twice, compare complete tarball bytes with `timingSafeEqual`, and record the exact npm integrity and SHA-256 values. Running it with any other Node or npm version must fail before packing.

## Tag and publication gate

After merge and only after explicit approval:

```bash
git tag -a v0.6.3 <approved-merged-recovery-commit-sha> -m 'feishu-md-sync v0.6.3'
git push origin refs/tags/v0.6.3
```

Approve the protected `npm` environment separately. The workflow must stop before publication if the runtime toolchain or either packed artifact differs from the committed manifest.

## Post-release verification

Run from a clean checkout detached at the immutable `v0.6.3` tag after registry propagation:

```bash
set -euo pipefail
REPO_ROOT="$(git rev-parse --show-toplevel)"
cd "$REPO_ROOT"
test -z "$(git status --porcelain)"
test "$(git rev-parse --abbrev-ref HEAD)" = 'HEAD'
test "$(git describe --tags --exact-match)" = 'v0.6.3'
TEMP_HOME="$(mktemp -d)"
CLI_PREFIX="$(mktemp -d)"
npm view feishu-docx-engine@0.1.1 version dist.integrity dist.shasum dist-tags --json
npm view feishu-md-sync@0.6.3 version dependencies dist.integrity dist.shasum dist-tags --json
npm install --prefix "$CLI_PREFIX" --ignore-scripts --no-audit --no-fund feishu-md-sync@0.6.3
```
